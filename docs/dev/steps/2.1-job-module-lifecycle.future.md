# Step 2.1 - Job Module and Worker Lifecycle

## Metadata

- Status: Planned
- Depends on: Step 1.8
- Estimate: 1 day

## Goal

建立独立 `ChikaJobs` Runtime 模块和由 EngineContext 管理的 Worker 生命周期，首版只执行无依赖任务。

## Responsibility

- JobSystem 拥有 Worker 线程、启动和停止。
- EngineContext 决定创建顺序和线程预算。
- Job callable 不拥有 JobSystem。

## Path Plan

- `engine/Runtime/Jobs/CMakeLists.txt`
- `Jobs/include/ChikaEngine/jobs/JobSystem.hpp`
- `Jobs/include/ChikaEngine/jobs/JobSystemCreateInfo.hpp`
- `Jobs/src/JobSystem.cpp`
- `engine/Runtime/CMakeLists.txt`
- `engine/include/ChikaEngine/EngineContext.hpp`
- `tests/unit/JobLifecycleTests.cpp`

## Implementation

- workerCount 默认 `hardware_concurrency - reservedThreads`，最少 1。
- Worker 稳定命名并注册 Profiler Thread。
- Initialize 失败回滚已创建线程。
- 首版提供简单 `Submit(function)` 和 Drain Shutdown。

## Unit Tests

- 1/2/4 Worker 启动和关闭。
- 1000 个无依赖任务全部执行一次。
- 重复 Initialize/Shutdown 行为明确。
- 部分 Worker 创建失败无泄漏。

## Delivery Checklist

- [ ] ChikaJobs 不依赖 Asset/Render/Editor。
- [ ] EngineContext 逆序关闭 Jobs。
- [ ] Worker 名称出现在 Profiler。
- [ ] 进程退出无后台线程。

## Boundaries

- 不实现 Handle、依赖、Work Steal。
- 不替换 `std::async`。
- 不访问 Vulkan。
