# Step 2.6 - ParallelFor and Deterministic Merge

## Metadata

- Status: Planned
- Depends on: Step 2.5
- Estimate: 1 day

## Goal

提供 Renderer 可直接使用的 Chunked ParallelFor 和按 Chunk Index 确定性合并工具。

## Responsibility

- ParallelFor 只拆分 index range。
- 调用方拥有每 Chunk 输出。
- Merge Helper 保证结果顺序，不解释业务对象。

## Path Plan

- `ParallelFor.hpp`
- `ParallelFor.cpp`
- `DeterministicChunks.hpp`
- `tests/unit/ParallelForTests.cpp`

## Implementation

- count、grain size、worker count 自适应。
- 小任务阈值自动串行回退。
- Chunk context 提供 begin/end/chunkIndex。
- 返回一个可等待 JobHandle。

## Unit Tests

- count 0/1、grain > count、非整除范围。
- 每个 index 恰好处理一次。
- 合并结果在 1/2/4/8 Worker 下 Hash 一致。
- Job 内调用 ParallelFor 不死锁。
- 异常在 join handle 上传播。

## Delivery Checklist

- [ ] 无共享 output vector push lock。
- [ ] 小范围串行阈值可配置。
- [ ] Chunk 数有上限。
- [ ] API 不依赖 Renderer 类型。

## Boundaries

- 不实现通用 parallel sort。
- 不承诺任务执行顺序。
- 只承诺合并结果顺序。
