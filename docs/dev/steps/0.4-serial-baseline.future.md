## Changed Files and Reasons

- `WindowDesc`、`GlfwWindow.cpp`：增加 VSync 与窗口可见性配置，避免桌面合成和窗口最小化污染采样。
- `Renderer`、`RenderDeviceContext`、`IRHIDevice`：传播 present 配置、frame-active 和 adapter 诊断信息。
- `VulkanRHIDevice.cpp`：可靠选择 GPU、选择非 VSync present mode、检查 descriptor reset 和跳过无 swapchain image 的录制。
- `ProjectDescriptor.cpp`：项目窗口配置可选择 `vSync`，默认仍为 true。
- `docs/dev/results/baseline/*`：保存不可手改的原始 JSON、硬件记录和串行分析。

## Knowledge and Design Notes

- **基线必须冻结环境**：Build 类型、GPU、分辨率、Present Mode、Camera、场景 Hash 和样本协议任一变化，结果都不能直接合并。
- **VSync Off 不等于一定无同步**：Vulkan surface 可能只支持 FIFO，因此实现会记录实际选择并对 fallback 发出警告。
- **CPU/GPU Bound 判断**：本次 50K 中 frame/engine-tick p50 约 38.8 ms，而 GPU p50 约 0.059 ms，瓶颈明确位于 CPU 场景同步、快照、剔除等路径，不是 GPU shading。
- **重复运行仍有噪声**：1K/10K 的绝对时间很短，调度噪声占比明显；报告保留全部三次结果，不挑选最好值冒充结论。
