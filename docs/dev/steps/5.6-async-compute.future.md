# Step 5.6 - Async Compute Queue Scheduling

## Metadata

- Status: Optional / Planned
- Depends on: Step 4.8 and proven queue overlap opportunity
- Estimate: 5+ days

## Goal

在设备存在独立 Compute Queue 且 Pass 可重叠时，让 RenderGraph 生成跨 Queue 调度与同步。

## Responsibility

- RHI 报告 queue families/capabilities。
- RenderGraph 编译 queue assignment、ownership transfer 和 semaphore dependency。
- Profiler 分离 Graphics/Compute GPU tracks。

## Path Plan

- `RHICapabilities.hpp`
- `IRHIDevice.hpp`
- `VulkanRHIDevice.hpp/.cpp`
- `RenderGraphQueueScheduler.hpp/.cpp`
- `RenderGraph.cpp`
- `tests/unit/RenderGraphQueueTests.cpp`

## Implementation

- Pass 声明 queue preference，不强制设备支持。
- 构建跨 queue dependency 和 timeline semaphore values。
- Queue family ownership transfer。
- 单 queue 设备回退原执行顺序。

## Unit Tests

- 双 queue DAG schedule 正确。
- 资源 hazard 产生跨 queue dependency。
- 单 queue fallback 与原顺序一致。
- resize/shutdown 等待正确 timeline value。

## Delivery Checklist

- [ ] GPU Timeline 显示真实 overlap。
- [ ] 无 queue ownership Validation Error。
- [ ] 有 overlap 前后 GPU frame 数据。
- [ ] 不支持设备行为稳定。

## Boundaries

- 没有独立 Queue 或重叠收益时不实施。
- 不与 Secondary Recording 同一提交完成。
- 不以更多 semaphore 数量冒充并行。
