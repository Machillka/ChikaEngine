## Goal

为 Visible Set、Packet、Batch 和 Draw 输入建立稳定 Hash，使现有串行路径成为后续并行/GPU 路径的正确性 Oracle。

## Responsibility

- Oracle 只观察纯数据结果。
- Renderer 继续使用原串行实现。
- Benchmark 保存 Hash 和 Counter。

## Implementation

- Visible Hash 基于稳定 RenderObjectHandle 顺序。
- Packet Hash 包含 pass/pipeline/material/mesh/object/depth quantization。
- Batch Hash 包含 key、first/count、instanced。
- Hash 算法和版本写入 Benchmark Result。

## Implementation Record

- `RenderValidation` 使用显式字段顺序的 FNV-1a，Visible Set 先按 Handle 排序，Packet/Batch 只写入 pass、资源 Handle、对象 Handle 和范围字段。
- `viewDepth` 按 `1/1024` 单位量化并定义非有限值与饱和规则；Hash 不读取地址、padding 或平台对象表示。
- Benchmark schema 升级为 v2，每帧保存 `hashVersion=1` 与 Visible/Packet/Batch/Draw Input 四类 Hash。

## Changed Files and Reasons

- `RenderValidation.hpp/.cpp`：建立串行和并行共用的纯数据正确性 oracle。
- `RenderDiagnostics.hpp`、`BenchmarkResult.hpp/.cpp`：把 Hash 和版本持久化到正式 Benchmark。
- `RenderPhase3Tests.cpp`：覆盖重复输入、插入顺序扰动和透明深度变化。

## Knowledge and Design Notes

- 正确性 Hash 是回归探针，不是资产身份或安全校验；版本变化必须显式更新，避免不同字段集合产生“可比较”假象。
- 对浮点值直接 Hash bit pattern 会放大不同编译器和计算路径的微小差异；量化把容许误差变成可审查契约。
