# Step 3.1 - Serial Renderer Correctness Oracle

## Metadata

- Status: Planned
- Depends on: Step 2.8
- Estimate: 1 day

## Goal

为 Visible Set、Packet、Batch 和 Draw 输入建立稳定 Hash，使现有串行路径成为后续并行/GPU 路径的正确性 Oracle。

## Responsibility

- Oracle 只观察纯数据结果。
- Renderer 继续使用原串行实现。
- Benchmark 保存 Hash 和 Counter。

## Path Plan

- `Render/include/ChikaEngine/RenderValidation.hpp`
- `Render/src/RenderValidation.cpp`
- `RenderVisibility.cpp`
- `RenderQueue.cpp`
- `tests/unit/RenderOracleTests.cpp`

## Implementation

- Visible Hash 基于稳定 RenderObjectHandle 顺序。
- Packet Hash 包含 pass/pipeline/material/mesh/object/depth quantization。
- Batch Hash 包含 key、first/count、instanced。
- Hash 算法和版本写入 Benchmark Result。

## Unit Tests

- 相同输入多次 Hash 相同。
- 插入顺序扰动在排序后得到相同 Hash。
- 单字段变化导致 Hash 变化。
- Transparent 深度顺序变化可检测。

## Delivery Checklist

- [ ] Serial 路径输出完整 Hash。
- [ ] Hash 不包含指针地址。
- [ ] 浮点量有稳定量化策略。
- [ ] Benchmark Schema 记录 hashVersion。

## Boundaries

- 不修改数据布局。
- 不并行任何代码。
- Hash 不用于 Runtime 资源身份。
