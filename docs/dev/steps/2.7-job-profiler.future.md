## Metadata

- Status: Complete (2026-06-20)

## Goal

把 Job 排队、执行、等待、Steal、Sleep 和 Worker Utilization 接入统一 Profiler。

## Responsibility

- Scheduler 发出通用 Profiler Event。
- Profiler 不调用 JobSystem。
- Editor 使用现有 Timeline lanes 展示 Worker。

## Path Plan

- `Jobs/src/JobProfiler.cpp`
- `Jobs/include/ChikaEngine/jobs/JobProfiler.hpp`
- `ProfilerTimelineModel.cpp`
- `tests/unit/JobProfilerTests.cpp`

## Implementation

- Job Zone 使用稳定 job name/category ID。
- Instant：enqueue、steal、cancel、failure。
- Counter：queue depth、active/sleeping workers、steal count。
- 区分 Queue Wait Duration 与 Execute Duration。

## Unit Tests

- 一个 Job 产生 queue/run/complete 配对事件。
- cancelled/failed Job 状态可见。
- Worker utilization 聚合不超过 100%。
- Profiler Disabled 不改变调度结果。

## Implementation Record

- Job callable 使用稳定 nameId 形成 Worker Zone；enqueue/start/complete/steal/cancel/failure instant 携带 `JobHandle::RawValue()` 作为因果 ID。
- Queue depth、active/sleeping Worker 和 queue-wait 作为 Counter；Editor Timeline 绘制 instant marker、payload tooltip 和最新 Counter。
- 新增独立 Release `ChikaJobBenchmark`，对比 serial、`std::async` 和 1/2/4/8 Worker。

## Changed Files and Reasons

- `JobProfiler.hpp/.cpp`、Profiler instant payload/trace export：建立调度因果与统计事件。
- `ProfilerTimelinePanel.hpp/.cpp`：显示 Worker instant 和 Counter，不让 Profiler 反向依赖 Jobs。
- `engine/Benchmark/src/JobBenchmark.cpp`、`docs/dev/results/jobs/`：保存调度粒度和扩展曲线证据。

## Knowledge and Design Notes

- 因果 ID 使用 generation handle 数值而非 callable 地址，避免泄露实现地址并能区分重用 Slot。
- Benchmark 通过 completion Job + condition variable 阻塞主线程，避免 Main Thread wait-help 污染 Worker 数定义。

## Verification Record

- Profiler enabled/disabled 调度结果一致；Worker 名称、Job Zone、causal payload 和 utilization 上限测试通过。
- Release 结果：1 us 不值得调度；100 us 在 8 Worker 下约 3.92x；1 ms 在 8 Worker 下约 7.05x。
