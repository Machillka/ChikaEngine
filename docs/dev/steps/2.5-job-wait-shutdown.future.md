## Goal

保证 Worker 内等待不死锁，并定义 External Wait、Drain Shutdown 和 Cancel Pending。

## Responsibility

- Worker Wait 继续执行可运行 Job。
- External Wait 使用条件变量阻塞。
- Shutdown Policy 决定接受、排空或取消任务。

## Implementation

- `Wait(handle)` 检测当前调用线程类型。
- Worker 采用 help loop，优先目标依赖但不破坏队列公平性。
- Shutdown 后拒绝新 Submit。
- Drain 完成已接收 Job；Cancel Pending 标记未运行 Job 并唤醒 waiter。

## Unit Tests

- 单 Worker 中 Parent 等 Child 不死锁。
- Worker 递归深度任务可完成。
- 多 External Waiter 同时唤醒。
- Drain 不丢任务；Cancel 不运行 pending callable。
- Shutdown 与 Submit 竞态返回稳定状态。
- Failed Job 在 Wait 时传播异常。

## Implementation Record

- Worker/Main Thread 的 `Wait` 使用 help-loop，External Wait 使用条件变量；单 Worker 内 Parent 等 Child 不死锁。
- `Drain` 拒绝新外部提交，但允许正在运行的 Job 创建后继工作；`CancelPending` 取消未运行 Job 并唤醒 waiter。
- 生命周期 shared mutex 让 Schedule/Wait/Release 与最终 Storage 重置互斥，避免 shutdown-submit 竞态访问已释放内存。

## Knowledge and Design Notes

- Wait-help 解决线程池内阻塞导致的饥饿死锁，但它也意味着主线程 `Wait` 可能执行任意 ready Job；主线程等待点必须受控。
- 不能强制终止 Running callable；CancelPending 只定义尚未运行工作的结果，Running Job 必须自行合作退出。
