# Step 2.5 - Wait-help, Drain and Cancellation

## Metadata

- Status: Planned
- Depends on: Step 2.4
- Estimate: 2 days

## Goal

保证 Worker 内等待不死锁，并定义 External Wait、Drain Shutdown 和 Cancel Pending。

## Responsibility

- Worker Wait 继续执行可运行 Job。
- External Wait 使用条件变量阻塞。
- Shutdown Policy 决定接受、排空或取消任务。

## Path Plan

- `JobWait.hpp/.cpp`
- `JobShutdownPolicy.hpp`
- `JobSystem.cpp`
- `tests/unit/JobWaitTests.cpp`
- `tests/stress/JobShutdownStressTests.cpp`

## Implementation

- `Wait(handle)` 检测当前调用线程类型。
- Worker 采用 help loop，优先目标依赖但不破坏队列公平性。
- Shutdown 后拒绝新 Submit。
- Drain 完成已接收 Job；Cancel Pending 标记未运行 Job 并唤醒 waiter。

## Unit Tests

- 单 Worker 中 Parent 等 Child 不死锁。
- Worker 递归深度任务可完成。
- 多 External Waiter 同时唤醒。
- Drain 不丢任务；Cancel 不运行 pending callable。
- Shutdown 与 Submit 竞态返回稳定状态。
- Failed Job 在 Wait 时传播异常。

## Delivery Checklist

- [ ] Wait 无无限 spin。
- [ ] Shutdown 有最大测试超时。
- [ ] Engine 初始化失败回滚可调用 Shutdown。
- [ ] 所有 waiter 在取消时结束。

## Boundaries

- Main Thread 等待不负责窗口事件泵，调用点必须受控。
- 不实现强制终止 Running Job。
- 不吞掉异常。
