## Goal

把线程事件聚合成不可变 Frame Capture，并维护固定长度历史和卡顿帧标记。

## Responsibility

- Aggregator 消费已完成 Buffer。
- Frame Capture 拥有展示所需数据副本。
- History 使用有界内存，不反向阻塞 Producer。

## Path Plan

- `ProfilerFrame.hpp`
- `ProfilerAggregator.hpp/.cpp`
- `ProfilerHistory.hpp/.cpp`
- `tests/unit/ProfilerAggregationTests.cpp`

## Implementation

- 按 Frame ID 和线程重建 Zone 层级。
- 保存 inclusive/exclusive duration、Counter 和 dropped events。
- 最近 300 帧 ring history。
- 支持手动 Pin Frame 和 configurable hitch threshold。

## Unit Tests

- 多线程乱序提交后 Frame 归属正确。
- Zone parent/child 和 exclusive 时间正确。
- 未闭合 Zone 被标记为 malformed，不导致崩溃。
- History 覆盖旧帧但保留 pinned frame。
- 内存上限稳定。

## Implementation Record

- Aggregator 按 frame/thread 排序事件并用 stack 重建嵌套 zone，计算 inclusive/exclusive，未闭合或错配事件标记 malformed。
- History 默认保留 300 个 `shared_ptr<const ProfilerFrameCapture>`，支持按 frame ID 查找、替换、pin 和 hitch threshold。

## Changed Files and Reasons

- `ProfilerFrame`、`ProfilerAggregator`：把可变事件流冻结成展示模型。
- `ProfilerHistory`：有界帧队列和 pinned 捕获分离，避免 Editor 持有 producer 数据。

## Knowledge and Design Notes

- immutable capture 是采集线程与 UI 的快照边界，Editor 延迟一帧读取不会与 producer 发生数据竞争。
- exclusive time 是 inclusive 减直接/间接子区间累计；malformed 时结果用于诊断，不应参与严格性能回归。

## Verification Record

- 单测覆盖 parent/child 时长、Counter、hitch、未闭合恢复、history 淘汰和 pinned 保留。
