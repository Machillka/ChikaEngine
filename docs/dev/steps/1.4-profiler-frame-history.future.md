# Step 1.4 - Frame Aggregation and History

## Metadata

- Status: Planned
- Depends on: Step 1.3
- Estimate: 1-2 days

## Goal

把线程事件聚合成不可变 Frame Capture，并维护固定长度历史和卡顿帧标记。

## Responsibility

- Aggregator 消费已完成 Buffer。
- Frame Capture 拥有展示所需数据副本。
- History 使用有界内存，不反向阻塞 Producer。

## Path Plan

- `ProfilerFrame.hpp`
- `ProfilerAggregator.hpp/.cpp`
- `ProfilerHistory.hpp/.cpp`
- `tests/unit/ProfilerAggregationTests.cpp`

## Implementation

- 按 Frame ID 和线程重建 Zone 层级。
- 保存 inclusive/exclusive duration、Counter 和 dropped events。
- 最近 300 帧 ring history。
- 支持手动 Pin Frame 和 configurable hitch threshold。

## Unit Tests

- 多线程乱序提交后 Frame 归属正确。
- Zone parent/child 和 exclusive 时间正确。
- 未闭合 Zone 被标记为 malformed，不导致崩溃。
- History 覆盖旧帧但保留 pinned frame。
- 内存上限稳定。

## Delivery Checklist

- [ ] UI 只能读取 const Frame Capture。
- [ ] Frame ID 不依赖 vector index。
- [ ] malformed/dropped 指标可见。
- [ ] 聚合器 Shutdown 可排空数据。

## Boundaries

- 不接入 GPU。
- 不实现 Editor Panel。
- 不计算跨运行 percentile。
