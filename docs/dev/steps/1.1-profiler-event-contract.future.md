# Step 1.1 - Profiler Event Contract

## Metadata

- Status: Complete (2026-06-19)
- Depends on: Step 0.4

## Goal

定义与 UI、Renderer、Job System 解耦的 CPU Profiler 事件、时钟和名称身份。

## Responsibility

- `ChikaProfiler` 只定义采集模型和会话。
- Core 提供单调时钟，不认识 ImGui 或 Vulkan。
- 事件 Payload 保持固定大小并可跨线程移动。

## Implementation

- 事件类型：FrameMark、ZoneBegin、ZoneEnd、Counter、Instant、ThreadName。
- 保存 timestamp、threadId、nameId、type、payload。
- Name ID 使用稳定进程内 Interning；Capture 保存 ID 到字符串表。
- 明确时间单位和序列化版本。

## Unit Tests

- Event trivially copyable 或满足固定移动契约。
- Timestamp 单调不倒退。
- 相同名称返回相同 ID，不同名称不冲突。
- 多线程名称注册结果唯一且可查询。

- 不实现宏、Buffer 或 UI。

## Implementation Record

- 新增独立 `ChikaProfiler` 静态库；事件采用 40-byte trivially-copyable record，时间统一为 `steady_clock` 纳秒。
- 名称通过进程内 interning 转为 `uint32_t nameId`，事件热路径不保存字符串，Schema 固定为 v1。

## Changed Files and Reasons

- `Runtime/Profiler/{CMakeLists.txt,ProfilerEvent,ProfilerClock,ProfilerName}`：建立与 Render、Editor、Jobs 无关的基础契约；仅 Trace 实现私有依赖 nlohmann JSON。
- 根 `CMakeLists.txt`：增加 `CHIKA_ENABLE_PROFILING`，统一控制编译期 instrumentation。

## Knowledge and Design Notes

- 固定大小事件使每线程 ring buffer 可以按值发布，不需要事件级动态分配。
- Name ID 只保证进程内稳定；可持久化 Trace 必须同时导出解析后的名称，不能把 ID 当跨运行资产身份。

## Verification Record

- `ChikaProfiler` 独立目标构建通过；静态断言验证事件大小和 trivially-copyable。
- 单测覆盖单调时钟、同名去重和多线程名称注册。
