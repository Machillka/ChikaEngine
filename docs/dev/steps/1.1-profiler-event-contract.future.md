# Step 1.1 - Profiler Event Contract

## Metadata

- Status: Planned
- Depends on: Step 0.4
- Estimate: 1 day

## Goal

定义与 UI、Renderer、Job System 解耦的 CPU Profiler 事件、时钟和名称身份。

## Responsibility

- `ChikaProfiler` 只定义采集模型和会话。
- Core 提供单调时钟，不认识 ImGui 或 Vulkan。
- 事件 Payload 保持固定大小并可跨线程移动。

## Path Plan

- `engine/Runtime/Profiler/CMakeLists.txt`
- `engine/Runtime/Profiler/include/ChikaEngine/profiler/ProfilerEvent.hpp`
- `engine/Runtime/Profiler/include/ChikaEngine/profiler/ProfilerClock.hpp`
- `engine/Runtime/Profiler/include/ChikaEngine/profiler/ProfilerName.hpp`
- `engine/Runtime/Profiler/src/ProfilerClock.cpp`
- `tests/unit/ProfilerEventTests.cpp`

## Implementation

- 事件类型：FrameMark、ZoneBegin、ZoneEnd、Counter、Instant、ThreadName。
- 保存 timestamp、threadId、nameId、type、payload。
- Name ID 使用稳定进程内 Interning；Capture 保存 ID 到字符串表。
- 明确时间单位和序列化版本。

## Unit Tests

- Event trivially copyable 或满足固定移动契约。
- Timestamp 单调不倒退。
- 相同名称返回相同 ID，不同名称不冲突。
- 多线程名称注册结果唯一且可查询。

## Delivery Checklist

- [ ] ChikaProfiler 可独立构建。
- [ ] 公共结构有大小静态断言。
- [ ] 不依赖 Render/Editor/Jobs。
- [ ] Event Schema 有版本号。

## Boundaries

- 不采集事件。
- 不实现宏、Buffer 或 UI。
- 不保存动态字符串到单个 Event。
