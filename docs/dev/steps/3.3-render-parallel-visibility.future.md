## Implementation Record

- `BuildVisibilitySerial` 保留为 oracle；`BuildVisibilityParallel` 用 `ParallelFor` 划分 index range，每个 chunk 独占 `VisibilityResult`。
- 主线程严格按 `chunkIndex` 合并 tested/visible/culled 和对象指针；执行顺序不参与输出顺序。
- 默认 grain 为 256、对象阈值为 2048；Job 不可用、低于阈值或并行异常会记录原因并重跑完整串行准备流程。

## Changed Files and Reasons

- `RenderVisibility.hpp/.cpp`：让 Snapshot 与 compact instance 共用相同可见性规则。
- `ParallelRenderVisibility.hpp/.cpp`：实现无共享 push 锁的对象级分块。
- `RenderSettings.hpp`、`RenderPreparation.cpp`：统一阈值、grain、模式和回退语义。

## Knowledge and Design Notes

- 确定性来自“独占 chunk 输出 + 有序 merge”，不是强制 Worker 按顺序执行。
- 当前 Visibility 单对象计算很轻，Release 中 4 Worker 的阶段耗时仍高于串行；并行价值主要来自与 Packet/Sort 统一调度边界，未来应考虑把多个阶段融合，减少屏障。

## Verification Record

- 4,096 对象在 1/2/4/8 Worker 下与 serial Visible Hash、count、shadow/layer 规则一致。
