# Step 6.2 - Correctness and Performance Regression Gates

## Metadata

- Status: Planned
- Depends on: Step 6.1
- Estimate: 1-2 days

## Goal

建立硬正确性 Gate 与审慎性能回归报告，避免性能噪声导致错误 CI 结论。

## Responsibility

- Correctness tests 硬失败。
- Performance tests 默认生成趋势和 warning。
- 专用稳定机器准备好后才启用阈值失败。

## Path Plan

- `tests/integration/RenderPathEquivalenceTests.cpp`
- `tests/cmake/PerformanceSmoke.cmake`
- `.github/workflows/` 或现有 CI 配置
- `tools/benchmark/compare_runs.py`
- `docs/dev/performance-regression-policy.md`

## Implementation

- Hash、image diff、Validation Error 为硬 Gate。
- p95 regression 使用重复运行、置信区间/容差。
- 保存最近基线 manifest 和 commit。
- PR 输出变化摘要，不覆盖批准基线。

## Unit Tests

- 人工 correctness mismatch 必须失败。
- 性能 fixture 在阈值内/外分类正确。
- 缺样本/硬件变化返回 inconclusive。
- 基线更新要求显式命令。

## Delivery Checklist

- [ ] Policy 说明何时 hard fail。
- [ ] CI artifact 包含原始结果。
- [ ] Debug tests 与 Release perf 分离。
- [ ] 无单次 FPS 阈值。

## Boundaries

- 不在共享云 Runner 上宣称精确性能回归。
- 不自动接受更快但错误的结果。
- 不因性能 warning 阻塞所有开发。
