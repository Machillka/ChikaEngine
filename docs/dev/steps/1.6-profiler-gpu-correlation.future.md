# Step 1.6 - GPU Timestamp Frame Correlation

## Metadata

- Status: Planned
- Depends on: Step 1.5, existing Vulkan timestamp queries
- Estimate: 1-2 days

## Goal

将延迟读取的 GPU Pass Timing 关联到原始 Frame Capture，而不是错误显示在当前帧。

## Responsibility

- RHI 产生 frame-tagged GPU timing。
- Render Adapter 把 GPU timing 转换为 Profiler GPU Track。
- Profiler 不读取 Vulkan Query Pool。

## Path Plan

- `RenderDiagnostics.hpp`
- `IRHIDevice.hpp`
- `VulkanRHIDevice.hpp/.cpp`
- `RenderGraph.cpp`
- `ProfilerGpuTrack.hpp/.cpp`
- `tests/unit/ProfilerGpuCorrelationTests.cpp`

## Implementation

- Timing 增加 absoluteFrameId、queue/track ID 和 valid flag。
- Fence 完成后解析旧帧 Query。
- Aggregator 将 GPU Pass 挂接到匹配 Capture。
- Query overflow、not-ready、disjoint/invalid 有明确状态。

## Unit Tests

- 模拟 N+2 返回 Timing，仍写入 Frame N。
- Query not-ready 不阻塞 CPU。
- Overflow 不产生越界 query。
- 相同 Pass 名在不同帧不混淆。

## Delivery Checklist

- [ ] 无 `WaitIdle()`。
- [ ] GPU Track 显示原始 Frame ID。
- [ ] Mock RHI 支持测试。
- [ ] Forward/Deferred 多帧运行无 Validation Error。

## Boundaries

- 首版只支持 Graphics Queue Track。
- 不做 CPU/GPU 时钟绝对校准。
- 不实现 Async Compute。
