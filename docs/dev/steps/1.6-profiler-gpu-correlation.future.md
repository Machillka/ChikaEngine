# Step 1.6 - GPU Timestamp Frame Correlation

## Metadata

- Status: Complete (2026-06-19)
- Depends on: Step 1.5, existing Vulkan timestamp queries
- Estimate: 1-2 days

## Goal

将延迟读取的 GPU Pass Timing 关联到原始 Frame Capture，而不是错误显示在当前帧。

## Responsibility

- RHI 产生 frame-tagged GPU timing。
- Render Adapter 把 GPU timing 转换为 Profiler GPU Track。
- Profiler 不读取 Vulkan Query Pool。

## Path Plan

- `RenderDiagnostics.hpp`
- `IRHIDevice.hpp`
- `VulkanRHIDevice.hpp/.cpp`
- `RenderGraph.cpp`
- `ProfilerGpuTrack.hpp/.cpp`
- `tests/unit/ProfilerGpuCorrelationTests.cpp`

## Implementation

- Timing 增加 absoluteFrameId、queue/track ID 和 valid flag。
- Fence 完成后解析旧帧 Query。
- Aggregator 将 GPU Pass 挂接到匹配 Capture。
- Query overflow、not-ready、disjoint/invalid 有明确状态。

## Unit Tests

- 模拟 N+2 返回 Timing，仍写入 Frame N。
- Query not-ready 不阻塞 CPU。
- Overflow 不产生越界 query。
- 相同 Pass 名在不同帧不混淆。

## Delivery Checklist

- [x] 无 `WaitIdle()`，query result 不使用 `WAIT_BIT`。
- [x] GPU Track 显示原始 Frame ID。
- [x] Mock/Session 测试覆盖 N+2 回填。
- [x] Query overflow、not-ready/invalid 有明确状态。

## Boundaries

- 首版只支持 Graphics Queue Track。
- 不做 CPU/GPU 时钟绝对校准。
- 不实现 Async Compute。

## Implementation Record

- `RenderPassGpuTiming` 增加 queue ID 和 valid；Vulkan 保留录制时 absolute frame ID，frame fence 后无等待读取旧 query slot。
- Application 将 RHI timing 转成后端无关 `ProfilerGpuTiming`；Session 按 frame ID 替换历史 capture 或暂存到待完成帧。
- overflow 输出一个 invalid 诊断 timing；not-ready/异常结果保留 pass 名但标记 invalid，不伪造时长。

## Changed Files and Reasons

- `RenderDiagnostics`、`VulkanRHIDevice/CommandList`：RHI 负责 query 生命周期和有效性。
- `Application.cpp`、`ProfilerSession.cpp`：只做后端无关适配与 frame correlation。

## Knowledge and Design Notes

- GPU query 的“读取帧”不是“发生帧”；必须携带录制时 frame ID，不能依赖 history 下标。
- 当前只有 duration correlation，没有 CPU/GPU clock calibration，因此 Timeline/Trace 只表达同帧 pass 顺序和时长。

## Verification Record

- 单测模拟 timing 在 frame capture 完成后到达，确认替换 frame N 而非当前帧。
- Query 分配有 512 项边界保护，overflow scope 使用 sentinel 保持嵌套栈平衡。
