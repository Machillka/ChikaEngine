# Step 5.3 - Clustered Light Culling

## Metadata

- Status: Optional / Planned
- Depends on: Step 4.8 and measured many-light bottleneck
- Estimate: 4-5 days

## Goal

使用 Compute 构建 Cluster Light Lists，替代每像素遍历全部光源，并保持 Forward/Deferred 共用数据契约。

## Responsibility

- RenderWorld 提供 Light Proxy Snapshot。
- Compute Pass 构建 cluster grid/index list。
- Lighting Shader 消费 reflection-driven buffers。

## Path Plan

- `Render/include/ChikaEngine/ClusteredLighting.hpp`
- `Render/src/ClusteredLighting.cpp`
- `Assets/Shaders/cluster_build.comp`
- Forward/Deferred lighting shaders
- `RenderPipelinePasses.cpp`
- `tests/unit/ClusterGridTests.cpp`

## Implementation

- 可配置 X/Y/Z cluster dimensions。
- View-space light bounds 和 cluster assignment。
- bounded index list、overflow counter 和 fallback。
- RenderGraph LightList Buffer dependencies。

## Unit Tests

- cluster index/depth slice 数学。
- 0 lights、单光源、超容量、屏幕边缘光源。
- Forward/Deferred 使用相同 light list contract。
- Overflow 不越界并可诊断。

## Delivery Checklist

- [ ] Many-light 基准显示 light count scaling。
- [ ] Reflection 无手写 binding。
- [ ] Cluster occupancy/overflow 可视化。
- [ ] 与全量循环有图像对比。

## Boundaries

- 不同时实现 Shadow Atlas。
- 不支持无限光源。
- 没有 many-light 数据时不启动本步骤。
