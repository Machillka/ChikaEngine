# Step 3.5 - Parallel Opaque Sort and Batch Build

## Metadata

- Status: Planned
- Depends on: Step 3.4
- Estimate: 2 days

## Goal

为 Opaque Packet 建立显式 Sort Key 和可验证并行排序；Transparent 保持稳定远到近排序。

## Responsibility

- Sort Key Builder 定义状态聚类语义。
- Sort 只重排 Packet，不访问 ResourceManager。
- Batch Builder 消费最终有序 Packet。

## Path Plan

- `RenderSortKey.hpp/.cpp`
- `ParallelRenderSort.hpp/.cpp`
- `RenderQueue.cpp`
- `tests/unit/ParallelRenderSortTests.cpp`

## Implementation

- Opaque key 包含 pass/pipeline/material/mesh/stable object tie-breaker。
- 首版可使用分块 sort + deterministic merge；Radix Sort 需测量后决定。
- Transparent 暂保留串行 stable_sort。
- Batch 结果继续使用现有规则。

## Unit Tests

- 大量重复 key 与极端 key。
- 1/2/4/8 Worker 结果和串行顺序一致。
- Batch Hash、count、instanced flag 一致。
- Transparent 顺序未改变。

## Delivery Checklist

- [ ] Sort Key 位宽和溢出有测试。
- [ ] 并行路径有小规模回退。
- [ ] Batch 构建不读未排序输出。
- [ ] 前后排序耗时有数据。

## Boundaries

- 不并行 Transparent Sort。
- 不因排序优化改变渲染语义。
- 不直接切换 GPU-driven。
