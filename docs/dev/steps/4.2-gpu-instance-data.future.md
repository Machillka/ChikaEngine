# Step 4.2 - GPU Instance and Draw Group Data

## Metadata

- Status: Planned
- Depends on: Step 4.1, Step 3.2
- Estimate: 2 days

## Goal

定义 CPU/GPU 共享的 Static Opaque Instance、Bounds、Draw Group 和 Indirect Argument 数据契约。

## Responsibility

- Render module 定义后端无关 Layout。
- ResourceManager 提供 Mesh index ranges 和 Material/Pipeline group identity。
- RHI 只接收 BufferDesc 和 DrawIndirect 参数。

## Path Plan

- `Render/include/ChikaEngine/GpuDrivenData.hpp`
- `Render/src/GpuDrivenData.cpp`
- `Resource/include/ChikaEngine/RenderResourceView.hpp`
- `Assets/Shaders/include/gpu_driven_layout.glsl`
- `tests/unit/GpuDrivenLayoutTests.cpp`

## Implementation

- `GpuInstanceData`：transform、bounds、drawGroupIndex、object ID。
- `GpuDrawGroup`：indexCount、firstIndex、vertexOffset、instance range。
- 明确 std430 对齐和矩阵布局。
- CPU 生成稳定 draw group order 和 layout version。

## Unit Tests

- C++ size/align/offsetof 与 shader side constants 一致。
- 相同输入生成相同 group order/hash。
- Static Opaque 分类准确。
- 空场景和超过容量返回明确结果。
