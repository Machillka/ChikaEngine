# Step 1.8 - Trace Export and Profiler Overhead Gate

## Metadata

- Status: Planned
- Depends on: Step 1.7
- Estimate: 1-2 days

## Goal

完成 Chrome/Perfetto Trace 导出，并量化 Profiler Disabled/Enabled 开销。

## Responsibility

- Exporter 只读取 Capture。
- Benchmark 负责开销实验。
- Profiler Gate 决定能否进入 Job System。

## Path Plan

- `ProfilerTraceExporter.hpp/.cpp`
- `tests/unit/ProfilerTraceExporterTests.cpp`
- `engine/Benchmark/src/ProfilerOverheadBenchmark.cpp`
- `docs/dev/results/profiler/`

## Implementation

- 输出 thread metadata、complete events、counters、GPU track。
- 写入 build/hardware/frame metadata。
- 原子写临时文件后 rename，失败不留下半文件。
- 对比 compile-disabled、runtime-disabled、enabled。

## Unit Tests

- JSON 可解析且事件时间单位正确。
- 特殊字符名称正确转义。
- 不完整 Capture 仍输出 dropped/malformed metadata。
- 文件写失败返回错误。

## Delivery Checklist

- [ ] Perfetto/Chrome 可打开 Trace。
- [ ] Disabled 目标开销 <0.1%。
- [ ] Enabled 目标开销 <2%。
- [ ] 保存原始运行结果和截图。

## Boundaries

- 目标未达到时记录真实值，不放宽测试条件。
- 不实现网络 Trace。
- 不 Hook Allocator。

## Gate

Profiler 可稳定用于后续 Job 和 Renderer 优化，且自身开销有数据说明。
