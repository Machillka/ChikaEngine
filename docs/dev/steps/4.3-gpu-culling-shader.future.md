## Knowledge and Design Notes

- 本阶段选择 atomic append 而不是 prefix sum，是为了先建立正确性通道。atomic append 成本会进入后续 benchmark，之后再用 prefix sum 或 wave/block compaction 优化。
- compute shader 反射的价值是让 set/binding、push constant 和 storage buffer 成为可检查契约，而不是把硬编码 descriptor 布局散落在 Renderer。
- 当前 production RenderGraph 中 compute pass 仍是 no-op contract pass，因为现有材质 shader 还没有通过 visible list 间接读取 object 数据。强行打开会得到错误画面或重复 CPU draw。
