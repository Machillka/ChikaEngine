## Gate

GPU-driven 请求可以进入真实 reset/cull/graphics consumer path；`state-diversity` 场景中 CPU oracle 与 GPU visible-list/indirect args 对比一致。Phase 4 Gate 的结论是 correctness 已打通，最终 Release 性能报告留到 Phase 6 的 benchmark matrix。

## Knowledge and Design Notes

- Gate 不能只看命令成功退出。这里必须检查 JSON：`requestedMode=gpu`、`effectiveMode=gpu`、`renderPathFallback=0`、`gpuDrivenValidationMatched=1` 同时成立才算 correctness gate 通过。
- 当前 Debug build frame p50 包含校验、资源更新和调试开销，不代表 renderer 稳态性能；正式报告需要 Release、多帧 warmup 和 1K/10K/50K/100K 对照。
- GPU-driven 的核心价值不是“调用了 compute shader”，而是把 CPU 生成 draw list 的一部分迁移到 GPU：GPU 写 visible list 和 indirect args，graphics pass 用 `DrawIndexedIndirect` 消费。
- Oracle 仍然保留 CPU path 作为正确性基准；这是做性能优化时避免“更快但画错”的关键工程约束。
