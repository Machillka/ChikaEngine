# Step 5.5 - RenderGraph Transient Memory Aliasing

## Metadata

- Status: Optional / Planned
- Depends on: Step 4.8 and measured transient memory pressure
- Estimate: 4-5 days

## Goal

利用 RenderGraph 已知资源生命周期复用兼容显存区间，降低 transient peak memory。

## Responsibility

- RenderGraph 编译器计算 lifetime 和 alias plan。
- RHI/Vulkan 分配器执行 compatible resource binding。
- Imported/Persistent 资源永不 alias。

## Path Plan

- `RenderGraphAllocator.hpp/.cpp`
- `RenderGraph.cpp`
- `RenderGraphResource.hpp`
- `RHIDesc.hpp`
- `VulkanRHIDevice.cpp`
- `tests/unit/TransientAliasingTests.cpp`

## Implementation

- Compatibility key：kind/size/alignment/format/usage/memory class。
- interval coloring/first-fit alias plan。
- alias boundary 插入必要 barrier。
- Debug Snapshot 显示 physical allocation ID 和 peak bytes。

## Unit Tests

- 不重叠兼容资源 alias。
- 重叠/不兼容/imported 资源不 alias。
- peak byte 计算准确。
- resize/graph recompile 回收安全。

## Delivery Checklist

- [ ] Graph Dump 显示 alias plan。
- [ ] Validation 无 image/buffer binding 错误。
- [ ] 显存 peak 有 Before/After 数据。
- [ ] 可关闭回退现有 pool。

## Boundaries

- 不同时重写完整 VMA allocator。
- 不 alias 跨帧 history。
- 无显存压力数据时跳过。
