# Step 4.4 - RenderGraph GPU-driven Buffer Flow

## Metadata

- Status: Planned
- Depends on: Step 4.3
- Estimate: 2 days

## Goal

通过 RenderGraph 表达 Instance Upload、Compute Visibility 和 Graphics Consumer 的 Buffer 依赖与状态转换。

## Responsibility

- RenderGraph 管理逻辑资源、依赖和 Barrier。
- ResourceManager/Upload Arena 拥有物理上传资源。
- Pass Execute 只录制命令。

## Path Plan

- `RenderGraph.hpp/.cpp`
- `RenderPassBuilder.hpp/.cpp`
- `RenderPipelinePasses.hpp/.cpp`
- `RenderGraphBlackboard.hpp`
- `tests/unit/GpuDrivenRenderGraphTests.cpp`

## Implementation

- Blackboard semantics：InstanceData、VisibilityFlags、VisibleInstances、IndirectArgs。
- Copy/Upload Pass 写 TransferDst。
- Compute Pass 读 StorageRead、写 StorageWrite。
- Graphics Pass 读 Vertex/Storage/IndirectArgument。
- Debug Snapshot 输出 Buffer lifetime 和 barrier。

## Unit Tests

- Pass 顺序 Upload -> Cull -> Graphics。
- 缺 producer/read-before-write 编译失败。
- StorageWrite -> IndirectArgument barrier 正确。
- 无 GPU-driven consumer 时相关 Pass 可被剔除。
