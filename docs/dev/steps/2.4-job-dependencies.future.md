# Step 2.4 - Completion Counters and Dependencies

## Metadata

- Status: Complete (2026-06-20)
- Depends on: Step 2.3
- Estimate: 2 days

## Goal

支持 Parent/Child、Fan-in/Fan-out 和 `ScheduleAfter`，只有依赖清零的 Job 才进入可运行队列。

## Responsibility

- Completion Counter 表达未完成工作数量。
- Dependency List 只持有稳定 Handle/内部引用。
- Scheduler 负责从 Waiting 转为 Queued。

## Path Plan

- `JobCounter.hpp`
- `JobDependency.hpp/.cpp`
- `JobSystem.cpp`
- `tests/unit/JobDependencyTests.cpp`

## Implementation

- Parent counter 在 child 完成后递减。
- 多前置依赖使用原子 remainingDependencies。
- 完成时发布 continuation，确保只 enqueue 一次。
- 失败/取消的依赖传播策略显式定义。

## Unit Tests

- Chain、Diamond、Fan-in 1000、Fan-out 1000。
- Parent 在全部 Child 后完成。
- 多 Worker 同时完成最后依赖只调度一次。
- 循环依赖在构建阶段拒绝或明确不支持动态环。
- 失败依赖按策略取消/运行 continuation。

## Implementation Record

- `ScheduleAfter` 使用 remaining-dependencies 原子计数，最后一个依赖完成时只发布一次 continuation。
- Parent 的 unfinished-work 同时计算自身和 Child；Parent 只有自身 callable 与全部 Child 完成后才进入终态。
- `Cancel`、`Propagate`、`RunAnyway` 三种依赖失败策略均显式实现。

## Changed Files and Reasons

- `JobStorage.hpp`：保存依赖、内部引用、Parent 和 completion counter。
- `JobSystem.cpp`：实现依赖注册、完成发布、Parent/Child 和异常传播。
- `tests/unit/JobSystemTests.cpp`：覆盖 Chain、Diamond、Fan-in 1000、Fan-out 和失败策略。

## Knowledge and Design Notes

- 依赖方持有的是内部引用，不是裸 Slot 指针；前置 Job 在 continuation 完成注册前不能被回收。
- 首版只支持调度时构建 DAG，不支持运行中任意加边，因此动态环检测不在当前 API 范围内。

## Verification Record

- Chain、Diamond、1000-way Fan-in、Worker fan-out 和三种失败策略测试通过。
