# Step 1.5 - Engine CPU Zone Integration

## Metadata

- Status: Planned
- Depends on: Step 1.4
- Estimate: 1 day

## Goal

在不改变行为的前提下，为主循环和关键 Subsystem 增加第一批稳定 Zone。

## Responsibility

- 各模块只声明自身高层 Zone 和 Counter。
- Profiler 不调用模块内部逻辑。
- Zone 名称作为分析 API，避免每次重命名。

## Path Plan

- `engine/src/Application.cpp`
- `engine/src/EngineContext.cpp`
- `Framework/src/scene/scene.cpp`
- `Framework/src/subsystem/{Render,Physics,Animation}Subsystem.cpp`
- `Render/src/{renderer,RenderVisibility,RenderQueue,RenderGraph}.cpp`
- `tests/integration/ProfilerEngineIntegrationTests.cpp`

## Implementation

- Frame、Application Update、Scene Tick、Fixed Tick。
- Physics、Animation、Render Bridge。
- Snapshot、Visibility、Packet Build、Sort/Batch、RenderGraph Compile/Execute。
- Counter：objects、visible、packets、batches、draws。

## Unit Tests

- 一个 Headless/Mock 帧包含预期顶层 Zone。
- Zone 顺序与生命周期一致。
- Profiling Off 时现有测试行为不变。
- 不允许单对象循环产生无界 Zone 数。

## Delivery Checklist

- [ ] Zone 粒度以阶段为主。
- [ ] Frame Counter 与 Render Statistics 一致。
- [ ] 无新跨模块依赖倒置。
- [ ] Debug/Release 均构建。

## Boundaries

- 不插桩每个 GameObject/Draw。
- 不优化任何阶段。
- 不接入 Jobs。
