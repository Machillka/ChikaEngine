# Step 2.8 - Asset Async Integration

## Metadata

- Status: Planned
- Depends on: Step 2.7
- Estimate: 1-2 days

## Goal

用 Job System 替换 AssetManager 零散 `std::async`，证明调度器能够服务真实引擎工作负载。

## Responsibility

- Asset Job 只执行磁盘读取、解析和 CPU 数据构建。
- AssetManager 保持缓存和 Handle 所有权。
- ResourceManager/GPU Upload 留在 Render/Main 线程边界。

## Path Plan

- `AssetManager.hpp/.cpp`
- `RenderResourceRequest.hpp`
- `EngineContext.cpp`
- `tests/integration/AssetJobIntegrationTests.cpp`

## Implementation

- AssetManager 注入 `JobSystem*`，不访问全局单例。
- `Load*Async` 返回引擎 Future/AssetRequest 或带状态 Handle。
- 相同 GUID 并发请求去重。
- Shutdown Drain/Cancel 与 AssetManager 生命周期一致。
- 完成 CPU Load 后发布主线程/GPU upload request。

## Unit Tests

- 并发请求同一资产只加载一次。
- 多资产并行完成且 Handle 有效。
- Loader 异常/缺失文件传播失败。
- Shutdown 期间请求不访问已销毁 AssetManager。
- GPU API 不在 Worker thread 调用。

## Delivery Checklist

- [ ] 删除 AssetManager 内直接 `std::async` 路径。
- [ ] Asset Job 出现在 Timeline。
- [ ] 同步 Load API 保持兼容或明确迁移。
- [ ] Asset Pipeline 现有测试全部通过。

## Boundaries

- 不实现完整 IO 专用线程池。
- 不并行 Shader/Pipeline Vulkan 创建。
- 不改变 GUID/meta/Cook 设计。

## Gate

Job System 通过正确性测试并成功驱动 Asset 工作负载，才能进入 Renderer 并行化。
