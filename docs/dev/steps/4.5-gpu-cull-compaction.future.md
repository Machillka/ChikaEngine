# Step 4.5 - GPU Frustum Culling and Visible Compaction

## Metadata

- Status: Planned
- Depends on: Step 4.4
- Estimate: 2-3 days

## Goal

在 Compute Pass 中完成 Frustum Culling，并把可见 Instance 压缩为按 Draw Group 可消费的列表。

## Responsibility

- Compute Shader 负责 bounds test、counter 和 compaction。
- CPU 负责上传 frustum、instance count 和 group metadata。
- Debug Readback 只用于抽样验证。

## Path Plan

- `gpu_frustum_cull.comp`
- 可选 `gpu_compact_visible.comp`
- `GpuDrivenVisibility.hpp/.cpp`
- `RenderPipelinePasses.cpp`
- `tests/integration/GpuVisibilityIntegrationTests.cpp`

## Implementation

- 第一版可使用单 pass atomic append；先正确再优化 prefix sum。
- 每 Draw Group 维护 visible count/offset 或稳定 global visible list。
- 清零 counter 通过明确 Copy/Compute Pass。
- objectCount/capacity 防越界。

## Unit Tests

- 小固定场景 CPU/GPU visible count 一致。
- 全可见、全剔除、边界 AABB、空场景。
- 超容量设置错误标志且不写越界。
- Validation Layer 无 storage/barrier 错误。

## Delivery Checklist

- [ ] 可见计数可显示。

- [ ] Counter 每帧正确 reset。
- [ ] 原子竞争成本进入 Profiler/Benchmark。

## Boundaries

- 不提交 Indirect Draw。
- 不做 Occlusion。
- 不承诺 atomic append 是最终算法。
