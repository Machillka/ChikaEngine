# Step 6.3 - Portfolio Artifacts and Technical Narrative

## Metadata

- Status: Planned
- Depends on: Step 6.2
- Estimate: 2-3 days

## Goal

把代码、测试和数据组织为招聘方可在数分钟内理解、并可进一步复现的技术作品集。

## Responsibility

- README 解释问题、架构、取舍、结果和限制。
- Capture/Trace/报告提供证据。
- 演示视频只展示技术闭环。

## Path Plan

- Root `README.md` portfolio section
- `docs/portfolio/architecture.md`
- `docs/portfolio/performance-report.md`
- `docs/portfolio/reproduction.md`
- `docs/portfolio/media/`
- `docs/portfolio/captures/manifest.json`

## Implementation

- 架构图：Profiler、Jobs、RenderWorld、RenderGraph、RHI。
- Job 状态机和 GPU-driven data flow 图。
- 3-5 页性能报告，引用原始 JSON。
- 60-90 秒视频：Timeline -> Worker scaling -> GPU mode -> results。
- 提供 Perfetto Trace 和 RenderDoc Capture manifest。

## Unit Tests

本步骤不新增 Runtime 单元测试；使用文档链接检查、结果 Schema 校验和干净环境复现实验作为自动验证。

## Verification

- 在干净机器按 reproduction 文档构建并运行最小 Benchmark。
- 所有文档链接和命令有效。
- Capture 标明 commit、GPU、driver、build config。
- 简历数字能追溯到报告和原始数据。

## Delivery Checklist

- [ ] 30 秒可看懂项目亮点。
- [ ] 5 分钟可理解核心设计。
- [ ] 30 分钟可复现实验。
- [ ] 明确列出未完成与适用边界。
- [ ] 英文简历 bullet 使用真实数据。

## Boundaries

- 不把普通场景漫游当技术演示主体。
- 不使用无法复现的宣传数字。
- 不隐藏低负载负收益和 fallback。

## Final Gate

作品集能够证明设计、实现、测试、性能分析和技术表达五个维度，而不是只展示功能截图。
