# Step 4.6 - Indirect Argument Generation and Submission

## Metadata

- Status: Planned
- Depends on: Step 4.5
- Estimate: 2-3 days

## Goal

由 GPU 生成每 Draw Group Indirect Arguments，并通过 RHI 执行实际 `DrawIndexedIndirect`。

## Responsibility

- Compute 生成参数和 instance range。
- RHI 负责后端命令与能力限制。
- Graphics Pass 绑定 group resources 并消费 indirect buffer。

## Path Plan

- `GpuDrivenSubmission.hpp/.cpp`
- `RenderPipelinePasses.cpp`
- `IRHICommandList.hpp`
- `VulkanCommandList.cpp`
- `tests/unit/IndirectArgumentTests.cpp`
- `tests/integration/GpuIndirectDrawTests.cpp`

## Implementation

- 参数布局匹配 Vulkan indexed indirect command。
- 无 indirect-count 时按固定 group count 提交，零 instance group 不绘制实际实例。
- 有能力时独立增量使用 indirect count。
- Pipeline/material/mesh group 绑定顺序稳定。

## Unit Tests

- Argument 字段和 stride 正确。
- 多 group/零 visible/单 instance。
- Mock RHI 收到预期 offset/count/stride。
- 不支持能力时回退 CPU path。

## Delivery Checklist

- [ ] GPU-driven mode 产生真实画面。
- [ ] Draw statistics 区分 direct/indirect。
- [ ] Buffer 生命周期覆盖 in-flight frame。
- [ ] Forward/Deferred 支持范围明确。

## Boundaries

- 首版只处理 Static Opaque。
- Transparent/Skinned 继续 CPU path。
- 不使用 Vulkan 调用绕过 RHI。
