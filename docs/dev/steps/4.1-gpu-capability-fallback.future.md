# Step 4.1 - GPU-driven Capability Contract and Fallback

## Metadata

- Status: Planned
- Depends on: Step 3.6
- Estimate: 1 day

## Goal

定义 GPU-driven 所需 RHI 能力查询、模式选择和显式回退，不假定所有设备支持相同 Indirect 特性。

## Responsibility

- RHI 报告能力，不选择 Renderer 策略。
- Renderer 根据设置和能力选择 Serial/Jobs/GPU。
- Benchmark 记录 requestedMode 和 effectiveMode。

## Path Plan

- `RHI/include/ChikaEngine/RHICapabilities.hpp`
- `IRHIDevice.hpp`
- `VulkanRHIDevice.cpp`
- `RenderSettings.hpp`
- `Renderer.hpp/.cpp`
- `tests/unit/RHICapabilityTests.cpp`

## Implementation

- 查询 compute、storage buffer、indirect draw、multi-draw、draw-indirect-count。
- 模式枚举 `SerialCpu/JobCpu/GpuDriven`。
- 不支持 GPU 路径时返回可诊断 fallback reason。
- 不允许静默切换后仍报告 GPU mode。

## Unit Tests

- Mock 能力矩阵选择正确模式。
- 缺 compute/indirect 时回退 Job/Serial。
- 用户要求 strict GPU mode 时返回失败。
- Benchmark 结果记录 effective mode。

## Delivery Checklist

- [ ] Capability 不暴露 Vulkan 类型。
- [ ] Editor 显示 fallback reason。
- [ ] 默认仍使用稳定 CPU 路径。
- [ ] Game/Editor 现有启动不变。

## Boundaries

- 不创建 GPU Buffer。
- 不编写 Compute Shader。
- 不要求 indirect count。
