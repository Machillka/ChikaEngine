# Step 1.7 - Editor Profiler Timeline

## Metadata

- Status: Planned
- Depends on: Step 1.6
- Estimate: 2 days

## Goal

实现只读 Timeline、Frame History、Thread Lanes、GPU Lane 和 Zone Details。

## Responsibility

- Editor Panel 只消费不可变 Capture。
- Timeline Model 负责缩放、筛选和布局，可脱离 ImGui 测试。
- Runtime Profiler 不依赖 Editor。

## Path Plan

- `engine/Editor/include/ProfilerTimelinePanel.hpp`
- `engine/Editor/src/ProfilerTimelinePanel.cpp`
- `engine/Editor/include/ProfilerTimelineModel.hpp`
- `engine/Editor/src/ProfilerTimelineModel.cpp`
- `tests/unit/ProfilerTimelineModelTests.cpp`

## Implementation

- 最近帧图、pinned/hitch 标识。
- Main/Worker/GPU lanes。
- Pan/Zoom、Zone hover/select、inclusive/exclusive details。
- Top Hotspots 和 dropped/malformed 警告。

## Unit Tests

- 时间到像素转换、裁剪和缩放稳定。
- 同线程嵌套 Zone lane depth 正确。
- 空 Capture、超长名称和大量 Zone 不崩溃。
- Panel 关闭不停止 Runtime 采集。

## Delivery Checklist

- [ ] 300 帧浏览可用。

- [ ] UI 不持有 Runtime 可变指针。
- [ ] 大 Capture 有可接受 UI 时间。

## Boundaries

- 不实现远程连接。
- 不在 OnImGuiRender 中聚合原始事件。
