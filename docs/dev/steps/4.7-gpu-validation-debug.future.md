## Changed Files and Reasons

- `GpuDrivenValidation.hpp/.cpp`：把正确性对比算法和延迟匹配队列独立出来，避免污染渲染热路径。
- `RenderStatisticsPanel.cpp`：暴露模式切换和 fallback 原因，便于调试平台能力或 consumer 初始化失败。
- `RenderDiagnostics.hpp`、`BenchmarkResult.cpp`：保存路径和 GPU-driven 诊断字段，保证 benchmark 可离线分析。
- `RenderPipeline.cpp`：把 readback/oracle 放在 ring slot 复用边界，避免当前帧强制 `WaitIdle()`。

## Knowledge and Design Notes

- GPU readback 不能在性能路径里每帧阻塞 map/wait；正确做法是延迟几帧读回，再用 frame id 找回当时的 CPU oracle。
- mismatch 报告用 object id，而不是截图。截图只能证明“看起来不对”，不能定位具体被错误剔除或错误绘制的对象。
- Editor 模式切换必须显示 effective path，否则用户选择 GPU 后实际跑 CPU 会误导调试和性能判断。
- 这里的 readback 依赖 frames-in-flight fence 保证 ring slot 完成；它适合 Debug/验证路径，不是最终的高性能 GPU readback 方案。
