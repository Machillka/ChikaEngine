# Step 4.7 - GPU-driven Validation and Debug Visualization

## Metadata

- Status: Planned
- Depends on: Step 4.6
- Estimate: 2 days

## Goal

建立 CPU Oracle 抽样对比、异步 Readback、模式切换和可见性调试，证明 GPU 结果正确。

## Responsibility

- Validation 模式可选，不污染性能模式。
- Readback 使用帧延迟，不阻塞当前 GPU。
- Editor 只显示结果和错误。

## Path Plan

- `GpuDrivenValidation.hpp/.cpp`
- `RenderDebugVisualization.hpp/.cpp`
- `RenderStatisticsPanel.cpp`
- `RenderSettings.hpp`
- `tests/unit/GpuVisibilityValidationTests.cpp`

## Implementation

- 每 N 帧抽样 CPU visible hash/count。
- GPU readback 返回 object IDs 或 compact hash/count。
- 显示 missing/extra IDs 和 frame latency。
- Editor 可切换 Serial/Jobs/GPU，显示 effective mode/fallback。

## Unit Tests

- 延迟 Readback 关联原始 Frame ID。
- missing/extra diff 算法正确。
- Validation Off 不创建 readback pass。
- Camera/Scene 切换清理 pending validation。

## Delivery Checklist

- [ ] 可生成 CPU/GPU mismatch 报告。
- [ ] 调试可视化区分 CPU/GPU culled。
- [ ] 无每帧 WaitIdle/map stall。
- [ ] Validation 与性能测试分离。

## Boundaries

- 不要求逐帧全量 Readback。
- 不用截图替代 ID/Hash 正确性。
- 不实现 Hi-Z Debug。
