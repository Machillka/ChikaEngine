# Step 3.4 - Parallel Render Packet Generation

## Metadata

- Status: Complete (2026-06-20)
- Depends on: Step 3.3
- Estimate: 2 days

## Goal

并行把 Visible Objects 转换为各 Pass Packet，同时保持资源读取线程安全和确定性合并。

## Responsibility

- Packet Task 只读取不可变 Material/Mesh Render Metadata。
- ResourceManager 在帧构建期间提供只读 Snapshot/View。
- 每 Chunk 生成本地 pass queues。

## Path Plan

- `RenderQueue.hpp/.cpp`
- `RenderResourceView.hpp/.cpp`
- `ParallelRenderQueue.hpp/.cpp`
- `ResourceManager.hpp/.cpp`
- `tests/unit/ParallelPacketTests.cpp`

## Implementation

- 先建立 ResourceManager 只读帧视图，禁止任务中触发 Upload。
- Chunk 输出 Shadow/Opaque/GBuffer/Transparent 本地 Packet。
- 按 chunkIndex 合并后再排序。
- 资源无效对象与串行路径采用相同策略。

## Unit Tests

- 各 Pass Packet Count 与串行一致。
- Packet Hash 在不同 Worker 数一致。
- Transparent viewDepth 计算一致。
- Resource reload 不与 active read view 竞态。

## Implementation Record

- `RenderResourceView::Build` 在 owner thread 从 ResourceManager 深复制本帧 Mesh 有效集合与 Material Pipeline Handle，Worker 只读该不可变视图。
- 每个 Packet chunk 构建本地 Shadow/Forward/GBuffer/Transparent Queue，之后按 chunk index 移动合并；无共享 `push_back` 和 GPU API。
- Resource View、Scene View 和 Snapshot 都活到全部 Job 完成；热重载只能影响下一帧新建视图。

## Changed Files and Reasons

- `ResourceManager.hpp/.cpp`：增加不抛异常的只读查询，视图构建时处理失效 Handle。
- `RenderResourceView.hpp/.cpp`：隔离 ResourceManager 的可变缓存与上传能力。
- `RenderQueue.hpp/.cpp`、`ParallelRenderQueue.hpp/.cpp`：拆开 Packet 生成、排序和 Batch 阶段。

## Knowledge and Design Notes

- 只读 C++ 方法不自动等于线程安全；这里通过“任务启动前深复制最小元数据”建立不变量，而不是让 Worker 直接读可能热重载的 SlotMap。
- Resource View 是帧级快照，不拥有 GPU 资源；实际资源生命周期仍由 ResourceManager/RHI 管理。
