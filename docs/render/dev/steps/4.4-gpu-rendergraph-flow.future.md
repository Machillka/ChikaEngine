## Goal

通过 RenderGraph 表达 Instance Upload、Compute Visibility 和 Graphics Consumer 的 Buffer 依赖与状态转换。

## Responsibility

- RenderGraph 管理逻辑资源、依赖和 Barrier。
- ResourceManager/Upload Arena 拥有物理上传资源。
- Pass Execute 只录制命令。

## Implementation

- Blackboard semantics：InstanceData、DrawGroups、VisibilityFlags、VisibleInstances、IndirectArgs、IndirectReset。
- Copy/Upload Pass 写 TransferDst。
- Reset Pass 将 CPU 初始化的 indirect args 真实 copy 到 GPU consumer 使用的 indirect buffer。
- Compute Pass 读 StorageRead、写 StorageWrite。
- Graphics Pass 读 Vertex/Storage/IndirectArgument。
- Debug Snapshot 输出 Buffer lifetime 和 barrier。

## Unit Tests

- Pass 顺序 Upload -> Reset -> Cull -> Graphics。
- 缺 producer/read-before-write 编译失败。
- StorageWrite -> IndirectArgument barrier 正确。
- 无 GPU-driven consumer 时相关 Pass 可被剔除。

## Implementation Record

- 新增 `AddGpuDrivenBufferFlow()`，用 RenderGraph 表达 InstanceData、DrawGroups、VisibilityFlags、VisibleInstances、IndirectReset 和 IndirectArgs 的生命周期。
- `RenderGraphSemantic` 增加 GPU-driven buffer key，并通过 blackboard 发布给后续 pass。
- `AddCopyPass()` 增加 `hasSideEffects` 参数，默认保持旧行为；GPU-driven upload/cull 在没有 consumer 时可被 graph 剔除。
- `RenderPipeline` 生产路径 import 当前 ring buffer slot，真实记录 `GPU Driven Reset`、`GPU Driven Frustum Cull` 和 GPU graphics consumer pass。

## Changed Files and Reasons

- `GpuDrivenRenderGraph.hpp/.cpp`：集中描述 GPU-driven upload/cull/consumer 的 graph contract。
- `RenderGraph.hpp/.cpp`：支持 copy pass 的副作用标记，避免无消费者时仍强制执行纯数据生产 pass。
- `RenderGraphBlackboard.hpp`：继续作为跨 pass resource lookup，不引入 Renderer 全局状态。
- `tests/unit/RenderPhase4Tests.cpp`：验证有 consumer 时顺序为 Upload -> Reset -> Cull -> Consumer，无 consumer 时 pass 可被剔除。

## Knowledge and Design Notes

- RenderGraph 的职责是“资源依赖与状态转换”，不是“业务策略”。GPU-driven pass 只声明读写，是否启用由 RenderPath selection 决定。
- 显式 `hasSideEffects=false` 可以让纯中间数据 pass 被剔除；present、overlay、真实 draw 等仍必须是 side-effect pass。
- `indirectArgs` 必须先 reset 再由 compute 原子增加 `instanceCount`。对于同一物理 buffer 的多阶段写入，使用 imported resource 明确告诉 RenderGraph 这是外部生命周期资源，而不是临时资源的重复 producer。
