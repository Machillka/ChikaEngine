# Step 5.4 - Parallel Secondary Command Recording

## Metadata

- Status: Optional / Planned
- Depends on: Step 4.8 and CPU command-recording bottleneck
- Estimate: 3-4 days

## Goal

在 Job System 中并行录制 Vulkan Secondary Command Buffers，并由主 Graphics Pass 按稳定顺序执行。

## Responsibility

- 每 Worker 拥有独立 per-frame Command Pool。
- RHI 暴露后端无关 secondary recording contract。
- RenderGraph 仍决定 Pass 顺序和资源 Barrier。

## Path Plan

- `IRHICommandList.hpp`
- `RHIDesc.hpp`
- `VulkanCommandList.hpp/.cpp`
- `VulkanRHIDevice.hpp/.cpp`
- `RenderGraph.cpp`
- `tests/unit/SecondaryCommandTests.cpp`

## Implementation

- Command Pool 按 frame/worker 隔离和 fence-safe reset。
- Secondary inheritance/dynamic rendering info 正确。
- Barrier 保留在 Primary；Secondary 只录制 Draw/Bind。
- 主线程按 packet chunk index 执行。

## Unit Tests

- Mock RHI secondary 顺序。
- Worker 不共享 Command Pool。
- resize/frame rollover 不提前 reset。
- fallback primary path 结果一致。

## Delivery Checklist

- [ ] Validation 无 command pool/recording 错误。
- [ ] Profiler 显示 worker recording zones。
- [ ] 有 command-heavy 场景收益数据。
- [ ] 低 draw count 自动回退。

## Boundaries

- 不把 RenderGraph Barrier 分散到 Worker。
- 不与 Async Compute 同时实施。
- 没有 CPU recording 瓶颈时跳过。
