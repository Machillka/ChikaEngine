## Implementation Record

- 新增 `GpuInstanceData`、`GpuDrawGroup`、`GpuIndexedIndirectCommand` 和 `GpuDrivenFrameData`，以 std430 友好的 16 字节对齐描述 static opaque 实例、draw group 和 indirect 参数。
- `BuildGpuDrivenFrameData()` 从 `RenderSceneView` 和 `RenderResourceView` 生成稳定输入：先按 pipeline/material/mesh 分组，再按 object id 排序，保证同一场景不同源顺序不会改变 hash。
- `RenderResourceView` 增加 mesh metadata，提供 index count、first index、vertex offset 和 index 格式，避免 GPU-driven 层回读 ResourceManager 内部对象。
- `Assets/Shaders/include/gpu_driven_layout.glsl` 保存 shader 侧共享布局，并通过 `.meta` 纳入 AssetDatabase 的只读扫描。

## Changed Files and Reasons

- `GpuDrivenData.hpp/.cpp`：集中定义 GPU-driven ABI、layout hash、visibility hash 和 indirect hash。
- `RenderResourceView.hpp/.cpp`：把 Mesh GPU metadata 复制进帧视图，Render 准备阶段不保留 ResourceManager 指针。
- `RenderDiagnostics.hpp`、`RenderPipeline.cpp`、`BenchmarkResult.cpp`：输出 GPU-driven instance/group/hash 统计。
- `Assets/Shaders/include/gpu_driven_layout.glsl(.meta)`：让 shader include 与 C++ 布局有明确版本边界。

## Knowledge and Design Notes

- GPU-driven 数据不能保存 CPU 指针或 SlotMap 句柄对象本体；shader 只需要稳定数值 ID 和索引范围。这样数据才可复制、排序、上传和延迟执行。
- 先建立 CPU 侧 deterministic builder 是必要的。它是后续 GPU compute/readback 的 oracle，能判断 GPU 结果错在 cull、compaction 还是 indirect args。
- `GpuIndexedIndirectCommand` 保持 20 字节，匹配 Vulkan indexed indirect ABI；为了跨后端，类型放在 Render 层，不包含 Vulkan 头。
