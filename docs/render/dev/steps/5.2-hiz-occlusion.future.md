# Step 5.2 - Conservative Hi-Z Occlusion Culling

## Metadata

- Status: Optional / Planned
- Depends on: Step 5.1
- Estimate: 3 days

## Goal

在 GPU Culling 中加入保守 Hi-Z 测试，并处理 camera motion、history latency 和 popping 风险。

## Responsibility

- Culling Shader 选择 mip 并测试 projected bounds。
- Renderer 决定 history 是否有效。
- Debug/Profiler 记录 occluded/retested 数量。

## Path Plan

- `gpu_frustum_cull.comp`
- `GpuDrivenVisibility.cpp`
- `RenderSettings.hpp`
- `RenderDebugVisualization.cpp`
- `tests/unit/HiZOcclusionMathTests.cpp`
- `tests/integration/HiZOcclusionTests.cpp`

## Implementation

- AABB/sphere 投影屏幕矩形。
- 根据 projected size 选择 conservative mip。
- bias、near-plane、off-screen、camera cut 降级可见。
- previous-frame result 不允许永久隐藏对象。

## Unit Tests

- 完全遮挡、部分遮挡、near-plane、屏幕外 bounds。
- Camera cut 后全部回退可见一帧。
- CPU reference occluder 场景与 GPU count 在容差内。
- 禁用时结果等于基础 GPU Frustum 路径。

## Delivery Checklist

- [ ] Occlusion counter/timing 可见。
- [ ] Debug overlay 可显示 occluded bounds。
- [ ] 高速相机无明显永久 popping。
- [ ] 高遮挡场景有真实收益数据。

## Boundaries

- 不承诺无任何 temporal popping。
- 不实现 software occlusion rasterizer。
- 数据无收益时不合入默认路径。
