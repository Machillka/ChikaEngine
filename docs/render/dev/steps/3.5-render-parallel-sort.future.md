## Goal

为 Opaque Packet 建立显式 Sort Key 和可验证并行排序；Transparent 保持稳定远到近排序。

## Responsibility

- Sort Key Builder 定义状态聚类语义。
- Sort 只重排 Packet，不访问 ResourceManager。
- Batch Builder 消费最终有序 Packet。

## Implementation Record

- `OpaqueRenderSortKey` 显式保存 pass/pipeline/material/mesh/object tie-breaker，不做易溢出的手工 bit packing。
- Opaque Queue 先并行 `stable_sort` 独立 run，再按固定二叉顺序 `inplace_merge`；最终有序后才串行构建 Batch。
- Transparent 保持串行、稳定、远到近排序；低于 4,096 packets 直接回退串行。

## Changed Files and Reasons

- `RenderSortKey.hpp/.cpp`：集中定义状态聚类和稳定 tie-break 语义。
- `ParallelRenderSort.hpp/.cpp`：实现可验证的 chunk sort 与 deterministic merge。
- `RenderQueue.cpp`：串行和并行复用同一 key 与 Batch 规则。

## Knowledge and Design Notes

- 首版选择 comparison sort，而不是 Radix Sort：Handle/key 语义仍在变化，当前先用 oracle 保证正确性；只有比较成本成为主要瓶颈后才值得固定 packed key 位宽。
- 50K Release p50 的 Sort 从 serial `9.7797 ms` 降到 4 Worker `4.9236 ms`；但 merge 和 Batch 仍串行，是 8 Worker 扩展回落的主要原因之一。
