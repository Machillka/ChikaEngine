## Goal

由 GPU 生成每 Draw Group Indirect Arguments，并通过 RHI 执行实际 `DrawIndexedIndirect`。

## Responsibility

- Compute 生成参数和 instance range。
- RHI 负责后端命令与能力限制。
- Graphics Pass 绑定 group resources 并消费 indirect buffer。

## Implementation

- 参数布局匹配 Vulkan indexed indirect command。
- `RecordGpuDrivenIndexedIndirect()` 在录制前检查 indirect/indexed/multi-draw 能力。
- 无 multi draw 能力时只允许单 command 提交。
- Pipeline/material/mesh group 绑定顺序由 CPU builder 保持稳定。
- `DrawGpuDrivenQueue()` 按 draw group 绑定 pipeline/material/mesh，然后对每组调用一次 `DrawIndexedIndirect`。
- `test.vert`/`skinned.vert` 通过 `visibleInstanceIndices[firstInstance + local]` 间接读取 `GpuInstanceData.model`。

## Knowledge and Design Notes

- Indirect draw 不是“少写一个 draw loop”这么简单，它要求 instance data、visible list、mesh/material binding、pipeline layout 和 buffer lifetime 都同步成立。
- 没有 `drawIndirectCount` 时，可以提交固定 group count，让 `instanceCount=0` 的 group 自然不画；但这仍需要每个 group 的 pipeline/material 绑定策略明确。
- 现有 Vulkan 设备能力报告中 `multiDrawIndirect=false`，所以 helper 会拒绝 commandCount > 1。这比静默调用后依赖 validation 报错更可控。
- 当前实现按 group 循环绑定资源并提交单条 indirect command，不是最终的 GPU material binning；它优先打通 correctness 和生命周期。
