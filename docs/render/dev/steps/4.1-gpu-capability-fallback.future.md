## Goal

定义 GPU-driven 所需 RHI 能力查询、模式选择和显式回退，不假定所有设备支持相同 Indirect 特性。

## Responsibility

- RHI 报告能力，不选择 Renderer 策略。
- Renderer 根据设置和能力选择 Serial/Jobs/GPU。
- Benchmark 记录 requestedMode 和 effectiveMode。

## Implementation

- 查询 compute、storage buffer、indirect draw、multi-draw、draw-indirect-count。
- 模式枚举 `SerialCpu/JobCpu/GpuDriven`。
- 不支持 GPU 路径时返回可诊断 fallback reason。
- 不允许静默切换后仍报告 GPU mode。

## Knowledge and Design Notes

- requested path 是用户意图，effective path 是本帧真实执行。成熟引擎通常也会区分 feature request、platform capability 和 runtime fallback，否则性能报告会误把 CPU 路径标成 GPU 路径。
- RHI capability 只回答“能不能”，不回答“该不该”。策略仍属于 Renderer，因为 Renderer 才知道 JobSystem、shader consumer 和场景数据是否可用。
- 当前 `multiDrawIndirect=false` 是有意保守：Vulkan 逻辑设备尚未启用对应 feature，因此 helper 会拒绝一次提交多个 indirect command。
