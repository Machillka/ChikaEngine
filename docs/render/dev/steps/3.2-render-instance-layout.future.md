## Implementation Record

- `RenderInstanceData` 是 16-byte 对齐、trivially-copyable 的 AoS；保存 transform、bounds、资源/对象 Handle、layer、flags 和 source index，不复制骨骼矩阵。
- `RenderSceneView` 持有 `shared_ptr<const RenderWorldSnapshot>`，并按 StaticOpaque、Skinned、Transparent、InvalidResource 连续分类；source index 安全映射回 Snapshot 对象。
- 当前选择 AoS，因为 Visibility 与 Packet 阶段会共同消费 Handle、Bounds、Flags 和资源字段；SoA 留给 SIMD 或 GPU 上传数据证明需要时再拆。

## Changed Files and Reasons

- `RenderInstanceData.hpp`：定义紧凑实例契约和静态对齐断言。
- `RenderSceneView.hpp/.cpp`：建立只读所有权、确定性分类与连续存储。
- `RenderPreparation.hpp/.cpp`：把 SceneView 作为串行/并行路径的共同输入。

## Knowledge and Design Notes

- 这相当于 Unreal `FScene`/Proxy 思路的简化版本：Gameplay 对象不进入 Renderer Worker，渲染数据通过稳定代理和帧视图传递。
- Release p50 显示旧 Snapshot 双 View 遍历为 `0.0067/0.0484/0.3454 ms`，SceneView 构建加双 View 遍历为 `0.1167/0.8413/3.9756 ms`（1K/10K/50K）。当前布局换来了并行边界，不是串行加速；后续应由 RenderWorld 增量维护持久实例数组以移除每帧重排。
