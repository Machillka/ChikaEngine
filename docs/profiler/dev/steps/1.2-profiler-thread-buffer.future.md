# Step 1.2 - Thread Registry and Event Buffer

## Metadata

- Status: Complete (2026-06-19)
- Depends on: Step 1.1
- Estimate: 1-2 days

## Goal

实现每线程固定容量写入 Buffer 和安全线程注册，移除热路径全局事件锁。

## Responsibility

- Thread Registry 管理线程身份和 Buffer 生命周期。
- Producer 只写当前线程 Buffer。
- Aggregator 是唯一 Consumer。

## Path Plan

- `ProfilerThreadRegistry.hpp/.cpp`
- `ProfilerEventBuffer.hpp/.cpp`
- `tests/unit/ProfilerThreadBufferTests.cpp`

## Implementation

- Thread-local registration token。
- Fixed-capacity 双缓冲或环形缓冲。
- Buffer 满时增加 droppedEventCount，不覆盖未消费完整事件。
- 线程退出发布完成区段；Profiler Shutdown 回收全部 Buffer。

## Unit Tests

- 单 Producer 顺序保持。
- 多线程各写 100K 事件无跨线程污染。
- 溢出不越界且 dropped count 正确。
- 线程反复创建/退出无悬空注册。
- Shutdown 与线程退出竞态可重复运行。

## Delivery Checklist

## Boundaries

- 不要求无锁 Work Stealing。
- 不聚合 Frame。

## Implementation Record

- 每个 producer 线程首次注册时获得稳定 ID、名称和固定容量 SPSC ring；后续写入只访问 TLS buffer。
- 满载时拒绝新事件并累计 dropped count，不覆盖 consumer 尚未读取的记录；线程记录由 registry 持有到进程结束。

## Changed Files and Reasons

- `ProfilerEventBuffer.hpp/.cpp`：固定容量、单 producer/单 consumer、release/acquire 发布协议。
- `ProfilerThreadRegistry.hpp/.cpp`：TLS 注册、线程退出标记、批量 drain 和容量配置。

## Knowledge and Design Notes

- SPSC 的成立条件是一个 buffer 只被所属线程写、只被 aggregator 读；Job System 后续必须保持这个所有权约束。
- overflow 选择“丢新事件”可保护已经发布的 Begin/End 顺序，但 Capture 仍必须显式显示 dropped，不能把不完整数据当精确结果。

## Verification Record

- 单测验证 4 个线程各写 100K 事件、顺序/线程隔离/退出标记、容量边界和 dropped count；Profiler shutdown/reinitialize 会先排空残留事件。
