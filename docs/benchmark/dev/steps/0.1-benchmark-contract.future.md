# Step 0.1 - Benchmark Target and Result Contract

## Metadata

- Status: Complete (2026-06-19)
- Depends on: Existing ChikaEngine and ChikaGame targets
- Estimate: 1 day

## Goal

建立独立 `ChikaBenchmark` 入口、稳定命令行和版本化结果 Schema，避免依赖 Editor 手动操作。

## Responsibility

- Benchmark target 负责选择场景、运行模式、预热、采样和结果输出。
- Engine 只暴露只读统计，不包含实验脚本。
- Result Writer 只序列化结构化结果，不计算 Renderer 行为。

## Implementation

- 支持 `--scene`、`--mode`、`--workers`、`--warmup`、`--frames`、`--output`。
- Result Schema 包含 schemaVersion、build、hardware、runConfig、samples、aggregates 和 counters。
- 非法参数、不可写输出、未知模式返回不同非零错误码。
- 首版允许使用空场景，只验证进程生命周期和结果格式。

## Implementation Record

- 新增独立 `ChikaBenchmarkCore` 与 `ChikaBenchmark`，由 `CHIKA_BUILD_BENCHMARKS` 控制，不依赖 Editor。
- CLI 严格解析场景、模式、worker、预热、采样、flush、分辨率、seed、超时和输出路径；Phase 0 明确拒绝尚未实现的 jobs/gpu 模式。

- `Application::OnFrameComplete` 提供通用帧完成观测点，Benchmark 不复制 Engine 主循环。

## Changed Files and Reasons

- `CMakeLists.txt`、`engine/CMakeLists.txt`、`engine/Benchmark/CMakeLists.txt`：建立可关闭的独立构建目标。
- `engine/Benchmark/include/ChikaEngine/benchmark/BenchmarkOptions.hpp`、`BenchmarkResult.hpp` 及对应 `.cpp`：固定 CLI 与版本化数据契约。
- `engine/Benchmark/src/main.cpp`：组合 EngineContext、Runner 和结果写入，不向 Runtime 反向注入 benchmark 依赖。
- `engine/include/ChikaEngine/Application.hpp`、`engine/src/Application.cpp`：增加默认 no-op 的帧完成 hook，保持 Game/Editor 行为不变。
- `tests/unit/BenchmarkTests.cpp`、`tests/CMakeLists.txt`：覆盖参数拒绝、Schema round-trip 与未知版本拒绝。

## Knowledge and Design Notes

- **独立进程基准**：避免 Editor UI、选中状态和手工操作成为隐含变量，也能用退出码接入 CI。
- **版本化 Schema**：性能数据是长期接口；读取器先检查 `schemaVersion`，防止旧工具把新字段误读成可比较结果。
- **观测点而非 benchmark 分支**：通用 `OnFrameComplete` 相当于应用层 instrumentation hook，Engine 不需要知道谁在消费指标。
- **原始样本优先**：聚合值可以重算，原始样本不可从 p95 反推，因此两者必须同时保留。
