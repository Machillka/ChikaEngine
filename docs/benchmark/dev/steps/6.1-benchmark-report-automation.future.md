# Step 6.1 - Benchmark Matrix and Report Automation

## Metadata

- Status: Planned
- Depends on: Step 4.8
- Estimate: 2 days

## Goal

自动运行场景/模式/Worker 矩阵，合并原始结果并生成可复核表格与图表数据。

## Responsibility

- Benchmark executable 产生单次结果。
- Runner script 只编排进程和合并结果。
- Report generator 不修改原始 JSON。

## Path Plan

- `tools/benchmark/run_matrix.py`
- `tools/benchmark/generate_report.py`
- `tools/benchmark/schema.py`
- `docs/dev/results/README.md`
- `tests/tooling/test_benchmark_report.py`

## Implementation

- Matrix：scene x serial/jobs/gpu x workers x repeat。
- 校验 build/hardware/scene hash 一致后聚合。
- 输出 CSV、Markdown tables 和 chart-ready JSON。
- 失败运行保留 stderr、exit code 和 partial manifest。

## Unit Tests

- 使用 fixture JSON 测试合并和 percentile。
- 配置 hash 不同拒绝比较。
- 缺失/失败 run 在报告中可见。
- 原始文件 checksum 不变化。

## Delivery Checklist

- [ ] 一条命令运行完整矩阵。
- [ ] 报告链接原始结果。
- [ ] 图表标明单位、硬件和样本数。
- [ ] Python 使用现有 uv 环境，不新增不明依赖。

## Boundaries

- 不在不稳定 CI 设置硬性能阈值。
- 不自动删除异常值。
- 不混合不同硬件结果。
