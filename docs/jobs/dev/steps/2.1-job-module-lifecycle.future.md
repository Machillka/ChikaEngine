## Metadata

- Status: Complete (2026-06-20)

## Goal

建立独立 `ChikaJobs` Runtime 模块和由 EngineContext 管理的 Worker 生命周期，首版只执行无依赖任务。

## Responsibility

- JobSystem 拥有 Worker 线程、启动和停止。
- EngineContext 决定创建顺序和线程预算。
- Job callable 不拥有 JobSystem。

## Implementation

- workerCount 默认 `hardware_concurrency - reservedThreads`，最少 1。
- Worker 稳定命名并注册 Profiler Thread。
- Initialize 失败回滚已创建线程。
- 首版提供简单 `Submit(function)` 和 Drain Shutdown。

## Implementation Record

- 新增独立 `ChikaJobs` 静态库；默认 Worker 数为 `hardware_concurrency - reservedThreads`，最少保留一个 Worker。
- `EngineContext` 负责创建、注入和逆序关闭 JobSystem；Asset、Renderer 都先于 JobSystem 销毁。
- Worker 创建中途失败会停止并 join 已创建线程，重复 Initialize/Shutdown 可用。

## Changed Files and Reasons

- `engine/Runtime/Jobs/`：建立不依赖 Asset、Render、Editor 的调度模块。
- `engine/Runtime/CMakeLists.txt`、`engine/include/ChikaEngine/EngineContext.hpp`、`engine/src/EngineContext.cpp`：统一线程预算与模块生命周期。

## Knowledge and Design Notes

- 线程池属于 Application/Engine 级服务，不应由 AssetManager 等业务模块隐式创建，否则线程预算和关闭顺序无法统一。
- `reservedThreads` 是预算策略而非硬件能力；显式 `workerCount` 始终优先，便于测试和性能复现。
