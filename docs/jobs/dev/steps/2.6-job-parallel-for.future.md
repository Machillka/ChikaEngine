## Goal

提供 Renderer 可直接使用的 Chunked ParallelFor 和按 Chunk Index 确定性合并工具。

## Responsibility

- ParallelFor 只拆分 index range。
- 调用方拥有每 Chunk 输出。
- Merge Helper 保证结果顺序，不解释业务对象。

## Path Plan

## Implementation

- count、grain size、worker count 自适应。
- 小任务阈值自动串行回退。
- Chunk context 提供 begin/end/chunkIndex。
- 返回一个可等待 JobHandle。

## Unit Tests

- count 0/1、grain > count、非整除范围。
- 每个 index 恰好处理一次。
- 合并结果在 1/2/4/8 Worker 下 Hash 一致。
- Job 内调用 ParallelFor 不死锁。
- 异常在 join handle 上传播。

## Implementation Record

- `ParallelFor` 根据 count、grain、Worker 数和最大 Chunk 数拆分范围，返回传播 Chunk 异常的 join Handle。
- 小范围或单 Worker 退化为一个 scheduled callable；部分提交失败会等待并释放已创建 Chunk。
- `MergeDeterministicChunks` 严格按 chunkIndex 合并，不承诺 Chunk 执行顺序。

## Changed Files and Reasons

- `ParallelFor.hpp`、`DeterministicChunks.hpp`：提供与 Renderer 类型无关的范围调度和确定性合并。
- `tests/unit/JobSystemTests.cpp`：覆盖 0/1/非整除范围、异常、Job 内调用和 1/2/4/8 Worker Hash。

## Knowledge and Design Notes

- 确定性来自输出分区和合并顺序，不来自线程执行顺序；每个 Chunk 写独占容器可避免共享 `push_back` 锁。
- Grain Size 是调度开销与并行度的折中，Release 数据表明当前有锁实现不适合约 1 微秒级碎任务。

## Verification Record

- 10,003 元素逐项一次、边界范围、异常传播、嵌套调用及 1/2/4/8 Worker 顺序 Hash 测试通过。
